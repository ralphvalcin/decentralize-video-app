Architecture — Final Shape

One ownership rule. Event-driven sync. PeerRecord with all agreed fields.

The Ownership Rule
Peer A (you)
✦ own isMuted, isCamOff
✦ own hasRaisedHand
✦ own isSpeaking
✦ own networkQuality
✦ own reaction, isAway
↓ emit on change
Socket.io
⇄
signaling
server
Peer B (remote)
⬇ cached isMuted
⬇ cached hasRaisedHand
⬇ cached isSpeaking
⬇ cached networkQuality
⬇ cached reaction, isAway
read-only in your store
You own your row. Others own theirs. PeerManager.tsx is the only bridge between Socket.io events and the store.
Final PeerRecord Shape
interface PeerRecord {
id: string
name: string
role: 'host' | 'guest'
// media
stream: MediaStream | null
isMuted: boolean
isCamOff: boolean
isScreenSharing: boolean
// connection
connectionState: RTCPeerConnectionState
networkQuality: 'good' | 'fair' | 'poor'
// interactions
isSpeaking: boolean
isPinned: boolean
hasRaisedHand: boolean
handRaisedAt: number | null
reaction: Reaction | null
isAway: boolean
}
Store Responsibilities (clean, no overlap)
usePeerStore
All per-peer data. One delete clears everything for a disconnected peer. The only store that touches PeerRecord.
useCallStore
Local peer only — your stream, your mute/cam state, roomId, screenSharePeerId. Nothing about remote peers.
useUIStore
Local UI only — panels open/closed, active modal, layout mode, toasts. No peer data ever touches this store.
useSessionStore
Room-level interactions — active poll, poll responses, pinned message, chat messages, recording state.
